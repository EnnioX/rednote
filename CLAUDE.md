# Circe 小红书自媒体项目规范

本项目为小红书自媒体账号 **Circe** 的内容生产仓库，聚焦 A 股趋势投资机会与全球资本市场信号。

- 账号名称：Circe洞察
- 简介：🌍 Capital Insight / 📈 每日追踪全球资本市场 / 🔍 寻找 A 股趋势投资机会
- 人格：神秘女性，冷静、克制、有洞察感，不喧哗、不喊单

---

## 1. 内容定位与原则

### 1.1 内容主线

- 主线：A 股趋势投资机会的发现与跟踪。
- 辅助：全球宏观流动性、产业周期、政策催化对 A 股的影响。
- 形式：每日一篇小红书图文笔记，源自系统化的方向扫描与深挖研究。

### 1.2 内容原则

- **结论先行**：先说“今天看到什么变化”，再给出依据。
- **证据说话**：不预测涨跌，只描述信号；聚焦“发生了什么”“意味着什么”“下一步验证什么”。
- **多空平衡**：每个方向必须主动寻找反方证据，不能只讲利好。
- **人话表达**：避免堆砌术语，关键数据必须解释为什么重要。
- **严格合规**：禁止具体买卖建议、目标价、承诺性表述；每篇发布稿必须附标准免责声明。

## 2. 内容生产流程

1. **信号扫描（每日）**基于当日市场、政策、产业、资金等七类信号，输出方向扫描报告。
2. **方向深挖（按需）**对扫描中优先级最高的 1–2 个候选方向，输出深挖报告，验证产业逻辑、A 股收入证据梯度、估值与拥挤度。
3. **发布稿生成（每日）**从当日扫描报告或最新深挖报告中，提炼 1 个最有传播价值的角度，生成小红书图文笔记。
4. **发布与归档**
   发布稿归档到 `posts/YYYYMMDD/`，原始报告保留在 `reports/YYYYMMDD/`。

### 4.1 命名规则

- 报告目录：`YYYYMMDD`（如 `20260619`），无连字符。

- 发布图片：平铺在 `posts/YYYYMMDD/` 目录下，使用 `YYYYMMDD-用途.png` 命名。SVG 源文件（可编辑）保留在 `assets/` 目录下。
- 数据快照：`RUN-YYYYMMDD-NNN.csv`，NNN 为当日运行序号。

## 5. 稿件 frontmatter 与元数据

### 5.1 扫描报告

```yaml
---
scan_id: SCAN-YYYYMMDD-NNN
scan_date: "YYYY-MM-DD"
data_cutoff: "YYYY-MM-DD HH:MM CST"
scan_window: "新信号 7 天（MM-DD ~ MM-DD），持续性回看 30 天，慢变量 90 天"
query_run_id: RUN-YYYYMMDD-NNN
status: candidate  # candidate / watching / validated / rejected
---
```

### 5.2 深挖报告

```yaml
---
direction_id: DIR-YYYYMMDD-NNN
parent_direction_id: ""  # 如由扫描报告衍生，填对应 scan_id
status: deep_research  # candidate / deep_research / validated / watching / rejected
fundamental_confidence: medium  # low / medium-low / medium / medium-high / high
mapping_confidence: medium
market_attractiveness: watch  # unknown / watch / attractive / avoid
data_cutoff: "YYYY-MM-DD HH:MM CST"
query_run_id: RUN-YYYYMMDD-NNN
data_snapshot: research/snapshots/RUN-YYYYMMDD-NNN.csv
previous_report: ""  # 上一期同名深挖报告路径
---
```

### 5.3 小红书发布稿（Word 格式）

发布稿为 `.docx` 文件，元数据写在正文第一页首行：

```
post_id: POST-YYYYMMDD-NNN | post_date: YYYY-MM-DD
source: reports/YYYYMMDD/XX报告.md
topic: 主题 | tags: 标签1, 标签2 | image_count: 4
```

### 5.4 状态定义

- `candidate`：刚发现，待验证。
- `deep_research`：正在深挖产业与 A 股映射。
- `watching`：逻辑成立，等待催化或买点。
- `validated`：有清晰催化窗口和 A 股标的，可进入观察名单。
- `rejected`：触发失效条件或证据不支持。

## 6. 证据与数据引用规范

### 6.1 证据 ID

- 格式：`EVD-YYYYMMDD-NNN`
- 每份证据记录：事件日期、发布日期、来源、等级、类型、主张、影响、立场、验证日期。
- 立场必填：`supporting` 或 `contradicting`；反方证据必须主动标注，不得省略。

### 6.2 证据等级

| 等级              | 说明                                               |
| ----------------- | -------------------------------------------------- |
| `primary`       | 一手来源：公告、官网、财报、监管文件、官方发布会   |
| `authoritative` | 权威机构：央行、工信部、头部券商研报、主流财经媒体 |
| `secondary`     | 二手解读：券商点评、行业媒体、自媒体，需交叉验证   |

### 6.3 数据引用格式（研究稿内部）

- 行内数据：`[DATA:RUN-ID:股票代码:指标=值]`例：`[DATA:RUN-20260619-004:688017.SH:pe_ttm=546.98]`
- 证据引用：`[EVD-YYYYMMDD-NNN]`，放在对应主张末尾。
- 报告末尾必须统一列出“证据记录”和“数据与来源”章节。

### 6.4 数据缺口记录

每份报告必须包含“缺口”章节，说明未获取的数据、对结论的影响、后续行动。不掩盖不确定性。

### 6.5 发布稿的来源表达

- 面向用户的发布稿不得展示 `EVD-*`、`DIR-*`、`RUN-*` 等内部编号。
- 正文必须提供“证据来源与主要内容”，使用“来源机构或公告名称 + 时间 + 核心内容”的自然语言格式。
- 只保留帮助读者判断可信度的信息，不暴露内部研究流程字段。

## 7. 小红书图文笔记转换规范

### 7.1 图文结构

- **发布稿 Word 仅含正文**，不嵌入任何图片；PNG 图片单独输出到 `posts/YYYYMMDD/`，按 `YYYYMMDD-用途.png` 命名。
- **固定 4 张图片**：封面 + 产业事实 + 供需盈利传导 + 催化时间轴，图片内容和规范参见 7.4。
- **封面图标题**：1 句钩子，≤20 字，点出变化而非结论。例：”人形机器人，订单第一次从 PPT 走进合同”。
- **图 1：已确认的产业事实**，用证据卡展示政策、量产、订单和关键数据。
- **图 2：产业供需与盈利传导**，用因果链展示需求如何经过产能和成本传导到利润，并标出瓶颈。
- **图 3：已识别催化**，用时间轴展示事件窗口及对应的可验证指标。
- **正文**：段落之间保持逻辑连续；每段 2–4 行，适合手机阅读。

### 7.2 正文格式

1. **开头**：一句话点出”今天最重要的变化”。
2. **中间**：2–3 个证据支撑，每个证据配一句话解释”为什么重要”。
3. **结尾**：给出”下一步看什么”或”关键风险”，不喊买卖。
4. **标签**：4–8 个，混用宽泛标签（#A股 #投资 #资本市场）和垂直标签（#人形机器人 #具身智能）。
5. **数据来源**：正文末尾增加”数据来源”区块，包含”发布日期”和”参考资料”两个字段。参考资料列出来源于原始深挖报告的证据清单，格式为”来源名称（日期）”。

### 7.3 发布稿 Word 文档结构

正文之后按以下顺序组织，每段之间空一行：

```
数据来源

发布日期：YYYY-MM-DD
参考资料：来源1（日期）；来源2（日期）；来源3（日期）...

#标签1 #标签2 #标签3 ...

⚠️ 本内容仅供信息分享与逻辑交流，不构成任何投资建议。股市有风险，决策需谨慎。
```

### 7.3 语气要求

- 保持 Circe 神秘女性人格：冷静、克制、有洞察感。
- 不用感叹号堆叠、不用“重磅”“炸裂”“必看”等夸张词。
- 人称以“我”或“我们看到”为主，不扮演机构或专家。

### 7.4 配图要求

- 封面必须有视觉主体：数据大字、关键词云或简洁信息图。
- 配色：深色/金色/紫色系，呼应神秘女性风格。
- 图片比例：小红书默认 3:4 或 1:1。
- 有深挖报告支撑时，默认输出 4 张图片：1 张封面和上述 3 张主题信息图；frontmatter 的 `image_count` 必须同步更新。
- 三张主题信息图统一使用 1080×1440 PNG，并保留同风格 SVG 源文件。
- PNG 输出到 `posts/YYYYMMDD/`，SVG 源文件保留在 `assets/`。命名统一为 `YYYYMMDD-cover`、`YYYYMMDD-industry-facts`、`YYYYMMDD-supply-profit`、`YYYYMMDD-catalysts`。
- 图片不得展示内部证据编号、方向编号或数据运行编号。
- 封面图统一导出为 PNG 格式，发布稿不得直接引用 SVG。
- 圆形头像与账号名 `Circe` 必须直接合成在封面 PNG 内部：头像位于核心数据下方，名字位于头像下方且清晰醒目。
- 封面 SVG 只包含文字和装饰元素，不包含头像。每次创建封面时的工作流：
  1. 先生成 SVG 文字版 → `rsvg-convert` 导出 PNG
  2. 用 `assets/circe-branding.png`（avatar + "Circe洞察"，380×330，透明底）合成到底部居中位置
  3. 使用 `PIL.Image.alpha_composite`，**不要裁剪透明通道**
  4. 保存到 `posts/YYYYMMDD-N/20-N-cover.png`
- ⚠️ 不要自己画头像圆圈或写 Circe 文字——用 `assets/circe-branding.png` 直接合成
- **信息图品牌标**：所有 industry-facts / supply-profit / catalysts PNG 的右上角叠加 `circe-branding.png`
  - 缩放至宽 280px（保持宽高比），位置距右边和上边各 40px（等距）
  - 叠加方式：`alpha_composite`，不裁剪透明通道

### 7.5 SVG 文字溢出防线
- 卡片可用宽度 **340px**（总宽 420 - 左右 padding 各 40）
- font-size 26 时：纯中文 ≤12 字/行，混合中英 ≤14 字符/行
- 超长文本必须拆成两行，卡片高度同步加高 30px
- 行内避免长数字+符号组合（如 "→ 50-100kW+"），优先拆分

## 8. 合规与风险提示

### 8.1 内容红线

- 禁止给出具体买入、卖出、加仓、减仓、目标价、止损位等投资建议。
- 禁止暗示“必涨”“稳赚”“确定性机会”等承诺性表述。
- 禁止在没有证据的情况下使用“内幕”“主力”“庄家”等词汇。
- 涉及具体标的时，必须同时说明风险，且风险不能流于形式。

### 8.2 标准免责声明

每篇发布稿正文末尾必须包含：

> ⚠️ 本内容仅供信息分享与逻辑交流，不构成任何投资建议。股市有风险，决策需谨慎。

### 8.3 风险提示位置

- 正文中涉及具体公司或方向时，在段落内或结尾处给出针对性风险。
- 风险提示必须具体，避免“市场有风险”这类空话。
  例：“当前绿的谐波 PE 547x，若量产进度不及预期，估值回调空间较大。”

### 8.4 法律与平台合规

- 不使用“推荐股票”“荐股”“跟单”等词汇。
- 不引导用户加入付费群、私域成交或转账。
- 引用数据标注来源，不伪造或夸大。

### 8.5 研究稿与发布稿区分

- `reports/` 内的扫描/深挖报告可更详细、更专业，但仍需遵守上述红线。
- `posts/` 内的发布稿面向公众，必须执行最严格的合规与风险提示。
